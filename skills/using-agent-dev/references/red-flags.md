# Red Flags — Stop When You're Rationalizing

| Thought                              | Reality                                                                                                    |
| ------------------------------------ | ---------------------------------------------------------------------------------------------------------- |
| "This is just a quick fix"           | Quick fixes become large tasks. Check for skills.                                                          |
| "I need more context first"          | Skills tell you HOW to gather context. Check first.                                                        |
| "Let me look at the code first"      | Skills tell you HOW to explore. Check first.                                                               |
| "I remember this skill"              | Skills evolve. Always read current version.                                                                |
| "The skill is overkill here"         | Simple tasks become complex. Last 3 redesigns started as "quick fixes". Check first — it costs 30 seconds. |
| "I'll just start then use the skill" | Skill check comes BEFORE the first action.                                                                 |
| "This doesn't need a formal process" | If a skill exists for it, use it.                                                                          |

## Cost of Skipping Process Skills

15 minutes of brainstorming prevents 2–4 hours of rework. Every "quick fix" that skipped design and required a redesign adds to this debt.

**Most common pattern**: skip brainstorming → build the wrong thing → rewrite from scratch.
